# Adversative-Antithesis Decomposition Codebook

Code `outputs/audit/adversative_antithesis_full_classical_coding.tsv`.

Use `yes`, `no`, or `uncertain` for the three coding columns. Leave
`uncoded` only for rows not yet inspected.

## classical_antithesis

Code `yes` when the two units form a classical antithesis-like pair:
opposed meanings are presented in comparable cola or comparable syntactic
slots.

Code `no` for scalar comparison alone, ordinary `instead of` or `rather
than` alternatives, concession, dialogue repair, fragmentary context, or
negation without balanced opposed presentation.

Code `uncertain` when the text is redacted, truncated, or needs more
context.

## semantic_opposition

Code `yes` when the two units encode opposed alternatives, properties,
states, choices, or scalar contrasts, even if the form is not balanced.

Code `no` when the relation is labelled adversative but the two units do
not clearly oppose each other.

## parallel_opposition

Code `yes` when the opposition is carried by matched grammatical frames,
slot substitution, or a clear paired construction.

Code `no` when only a contrast marker connects structurally unlike spans.

## antithesis_notes

Use short notes only where useful, for example `redacted`, `scalar_only`,
`instead_of_alternative`, `not_x_but_y`, `balanced_slots`,
`needs_context`, or `fragment`.
